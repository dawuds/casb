# ISO/IEC 27017:2015 — CASB control mapping (framework page)

> Code of practice for information security controls based on ISO/IEC 27002 for cloud services. Designed as the bridge between general infosec (27002) and cloud-specific (27017) controls. Often the easiest international standard to cite for CASB capabilities; reference cleanly cites the clause + version.

## Status

This page is a **framework**, not the full clause-by-clause mapping. The full clause mapping is a separate compliance deliverable.

## Why ISO 27017 matters for CASB

ISO 27017 sits on top of ISO 27002 (the controls catalogue) and adds cloud-specific guidance. Its CLD.* clauses are cloud-specific; the non-CLD clauses reference back to 27002 with cloud-specific implementation guidance.

For CASB:
- The cloud-customer / cloud-service-provider responsibility split (CLD.6.3.1) is foundational
- Monitoring of cloud services (CLD.12.4.5) maps directly to CASB activity logging
- Shared admin operational security (CLD.12.1.5) relates to OAuth-app governance
- Removal of cloud assets on exit (CLD.8.1.5) maps to vendor exit strategy

Auditors often accept ISO 27017 control IDs as cleaner-cited than country-specific regulator clauses (which may be confidentially supervised or under revision). Reference 27017 + the country-specific instrument together.

## Key CLD clauses with CASB relevance

> **The exact text and numbering should be verified against the published standard.** This is orientation, not a substitute for reading the document.

| Clause | Subject | CASB relevance |
|---|---|---|
| **CLD.6.3.1** | Shared roles and responsibilities within a cloud computing environment | Customer-responsibility section of the shared-responsibility model; CASB sits in the customer-responsibility scope for SaaS-access security |
| **CLD.8.1.5** | Removal of cloud service customer assets | Vendor exit strategy; data export on contract termination; documented policy-export and log-export capabilities |
| **CLD.9.5.1** | Segregation in virtual computing environments | Less direct CASB relevance (more SaaS-internal); CASB API connector consumes events at the SaaS boundary |
| **CLD.9.5.2** | Virtual machine hardening | IaaS-relevant; less direct for the SaaS-access slice CASB covers |
| **CLD.12.1.5** | Administrator's operational security | Privileged-access patterns; OAuth-grant governance for admin-permission classes |
| **CLD.12.4.5** | Monitoring of cloud services | **Direct CASB mapping** — discovery, activity logging, anomaly detection, audit-trail integrity, retention |
| **CLD.13.1.4** | Alignment of security management for virtual and physical networks | Less direct; SSE composition might reference this for the network-security alignment |

## Citation discipline

| Requirement | Detail |
|---|---|
| **Cite clause + standard version** | "ISO/IEC 27017:2015 CLD.12.4.5" — never "ISO 27017" without the colon-and-clause |
| **Quote verbatim from the published standard** | Don't paraphrase regulatory language; the standard is purchased / licensed for a reason |
| **Note any supplementary local interpretations** | E.g. BNM RMiT often references international standards including 27017; cite both |
| **Confirm against published standard, not third-party summaries** | Vendor or consulting summaries are secondary |

## Composition with other standards

ISO 27017 is one piece of the international-standards stack:

| Standard | Role |
|---|---|
| **ISO/IEC 27001:2022** | The ISMS standard; the certification basis; Annex A controls |
| **ISO/IEC 27002:2022** | The controls catalogue; detailed control descriptions |
| **ISO/IEC 27017:2015** | Cloud-specific overlay on 27002 |
| **ISO/IEC 27018:2019** | Cloud-PII-processor specific (companion to 27017) |
| **ISO/IEC 27701** | PIMS (Privacy Information Management System) overlay |
| **ISO/IEC 27036** | Supplier relationships (vendor-side; relevant for CASB vendor due diligence) |

For a regulated FI, citing 27001 (certification base) + 27017 (cloud overlay) + 27018 (PII processor overlay) is the typical structure.

## What an auditor will probe

| Question | Evidence required |
|---|---|
| Does the CASB vendor have ISO 27017 attestation in scope? | Vendor's published certificate / attestation from Trust Center |
| For CLD.6.3.1 (shared responsibility), where is the customer / vendor split documented? | Vendor's shared-responsibility model document; customer's implementation against it |
| For CLD.12.4.5 (monitoring), what activities are logged and how long are they retained? | CASB activity log + retention configuration; SIEM forwarding proof |
| For CLD.8.1.5 (removal of assets on exit), what is the exit / portability strategy? | Documented exit-cost estimate + vendor contract clauses |
| For CLD.12.1.5 (administrator operational security), how are admin OAuth grants governed? | Per [`../03-policies/`](../../03-policies/) `oauth-grant-restriction.md` policy + evidence |

## Standard version drift

ISO 27017:2015 has been the published version for over a decade. A revision is anticipated but not yet published as of writing. When the new version lands:
- Re-verify mappings against the new clauses
- Some CLD.* numbering may change
- Cite the new version + date
